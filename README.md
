# GitHub Release Tracker for Home Assistant

A Home Assistant integration that tracks GitHub releases and fires events when new releases are detected. Perfect for automating updates of GitHub-hosted applications.

## Features

- 🔔 **Event-based notifications** - Fires `github_release` event when new releases are detected
- 🔍 **Asset filtering** - Filter release assets by file type (e.g., `.apk`, `.zip`)
- 💾 **State persistence** - Remembers last checked release between restarts
- ⏰ **Intervals** - Check for updates once a day (default) or customize it in const.py 
- 📱 **Multi-repo support** - Track multiple GitHub repositories

## Installation

### HACS (Recommended)
1. Add this repository as a custom repository in HACS
2. Search for "GitHub Release Tracker" in HACS
3. Click "Install"
4. Restart Home Assistant

### Manual
1. Copy the `custom_components/github_release_tracker` folder to your `custom_components` directory
2. Restart Home Assistant

## Configuration

### UI Configuration (Recommended)
1. Go to **Settings** → **Devices & Services** → **Add Integration**
2. Search for "GitHub Release Tracker"
3. Enter the GitHub API URL (see format below)
4. (Optional) Add an asset filter (e.g., `.apk` for Android apps)

### GitHub API URL Format
Convert your GitHub repository URL to the API format:

| Repository URL | API URL |
|----------------|---------|
| `https://github.com/owner/repo` | `https://api.github.com/repos/owner/repo/releases/latest` |
| `https://github.com/chrisuthe/SendspinDroid` | `https://api.github.com/repos/chrisuthe/SendspinDroid/releases/latest` |

## Usage

### Event Data
When a new release is detected, the integration fires a `github_release` event with the following data:

```yaml
repo_name: "owner/repo"
release_version: "1.2.3"
release_name: "Awesome Release v1.2.3"
download_url: "https://github.com/owner/repo/releases/download/1.2.3/app-1.2.3.apk"
release_url: "https://github.com/owner/repo/releases/tag/1.2.3"
published_at: "2024-03-03T12:00:00Z"
asset_name: "app-1.2.3.apk"
asset_size: 15000000
author: "owner"
prerelease: false
body: "Release notes in markdown format..."
```

### Automation Example
```yaml
automation:
  - alias: "Notify on new GitHub release"
    trigger:
      platform: event
      event_type: github_release
    action:
      - service: notify.mobile_app
        data:
          title: "New Release: {{ trigger.event.data.release_name }}"
          message: >
            {{ trigger.event.data.repo_name }} v{{ trigger.event.data.release_version }}
            is now available!
          data:
            url: "{{ trigger.event.data.download_url }}"
```


## Asset Filtering
Use the asset filter to select specific file types from releases with multiple assets:

| Filter | Selects |
|--------|---------|
| `.apk` | First asset containing `.apk` in filename |
| `.zip` | First asset containing `.zip` in filename |
| (empty) | First asset in the release |

## Configuration Options

| Option | Required | Default | Description |
|--------|----------|---------|-------------|
| Repository URL | Yes | - | GitHub API URL for `/releases/latest` endpoint |
| Asset Filter | No | (empty) | Filter assets by filename (case-insensitive) |

## Troubleshooting

### Common Issues

1. **"Invalid GitHub repository URL" error**
   - Ensure you're using the API URL format
   - Check that the repository exists and has releases
   - Verify you have internet connectivity to GitHub

2. **No events firing**
   - Check Home Assistant logs for errors
   - Verify the integration is enabled
   - Ensure the repository has public releases

3. **Wrong asset selected**
   - Use a more specific asset filter
   - Check the release has assets uploaded

### Logs
Check Home Assistant logs for detailed information:
```yaml
logger:
  default: info
  logs:
    custom_components.github_release_tracker: debug
```
